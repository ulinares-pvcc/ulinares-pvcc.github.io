Chapter 7
Software Engineering
Term Definition
Software Engineering The application of principles used in the field of engineering, which usually deals with physical systems, to the design, development, testing, deployment and management of software systems.
Integrated Development Environment (IDE) The combine tools for developing software (editors, compilers, debugging tools, and so on) into a single, integrated package.
Association for Computing Machinery (ACM):
  1. purpose of ACM
  2. benefits of ACM membership
  3. student ACM membership
  1. Advancing the art, science, engineering, and application of computing, serving both professional and public interests by fostering the open exchange of information and by promoting the highest professional and ethical standards.
  2. Network of nearly 100,000 highly dedicated student and professional peers. Find targeted job opportunities. Gain hands-on experience and knowledge of the marketplace.
  3. Basic is $19. Basic plus Digital Library is $42. Basic plus Digital Library and Print CACM Magazine is $62.
Institute of Electrical and Electronics Engineers (IEEE):
  1. purpose of IEEE
  2. benefits of IEEE membership
  3. student IEEE membership
  1. Foster technological innovation and excellence for the benefit of humanity.
  2. Network with other technology professionals. Establish a professional profile highlighting your accomplishments.
  3. Must carry at least 50% of a normal full-time academic program as a registered undergraduate or graduate student in a regular course of study in IEEE designated fields. The total cumulative period for a member to hold the Student member grade and/or the Graduate Student member grade is limited to eight years.
Software Life Cycle: Development phase
  1. Requirement analysis
  2. Design
  3. Implemnetation
  4. Testing
  1. Specify what services the proposed system will provide. Identify any conditions (time constraints, security) on those services.
  2. Creating a plan for the construction of the proposed system.
  3. Writing of programs, creation of data files, and development of databases.
  4. One segment in the process for quality assurance.
Modular Implementation A software design technique that emphasizes separating the functionality of a program into independent, interchangeable modules, such that each contains everything necessary to execute only one aspect of the desired functionality.
Unifed Modeling Language (UML) A general-purpose modeling language in the field of software engineering.
Use Case A written description of how users will perform tasks on your website.
Use Case Diagram Describes the high-level functions and scope of a system.
Data relationships:
  1. one-to-one
  2. one-to-many
  3. many-to-many
  1. Association multiplicity where each patient is associated with only one room and each room is associated with only one patient.
  2. One physician is associated with many patients and each patient is associated with one physician.
  3. Each physician is associated with many patients and each patient is associated with many physicians.
Design pattern Provides a general reusable solution for the common problems that occur in software design.
Software quality assurance (SQA) The ongoing process that ensures the software product meets and complies with the organization's established and standardized quality specifications.
Software testing:
  1. glass box testing
  2. black box testing
  3. beta testing
  4. alpha testing
  1. A testing technique that examines the program structure and derives test data from the program logic/code.
  2. Testing a system with no prior knowledge of its internal workings.
  3. An opportunity for real users to use a product in a production environment to uncover any bugs or issues before a general release.
  4. The initial phase of validating whether a new product will perform as expected.
Documentation:
  1. user documentation
  2. system documentation
  3. technical documentation
  1. The content you provide end users with to help them be more successful with your product or service.
  2. Describe the software’s internal composition so that the software can be maintained later in its life cycle.
  3. A generic term for the classes of information created to describe (in technical language) the use, functionality or architecture of a product, system or service.
Intellectual property law
  1. software license
  2. software disclaimer
  3. best defenses against software liability claims
  1. A document that provides legally binding guidelines for the use and distribution of software.
  2. A software disclaimer is the software developer's formal notice to the user of its software that the developer is denying any and all liability for any damages arising out of using the software as well as denying any implied warranties.
  3. Apply sound software engineering principles during the software’s development. Use a level of care compatible with the software’s application. Produce and maintain records that validate these endeavors.

Program structures:

  • Sequential structure: Top-down execution
  • Decision structures: if/then/else
  • Iterative (Looping):
    • while loop: continue looping as long something is true
    • repeat/until loop: continue looping until something is true
    • for loop: loop a given number of times
  • Recursive structures:
    • Repeat a set of instructions as a subtask of itself
    • Repetition continues until:
      • degenerative case, which is a termination condition is met
      • (degenerative case is also called a "base case")

Two classic programming/database problems

  • Sorting: Descending order/Ascending order
    • Sorting numbers
    • Sorting strings (in accordance with ASCII values)
  • Searching: find specific data that meet a certain criteria
    • In database:
      • Filters: temporary search
      • Queries: search criteria are saved so they don't have to be recreated
  • Two popular search techniques:
    • Sequential search: search from top to bottom or bottom to top, one item at a time
    • Binary search:
      • only works if data is SORTED first
      • starts in the middle, then go to half of that, etc.

Two classic program goals:

  • Efficiency: Program takes minimal time and/or resources
  • Correctness: Program runs in expected ways and works for all expected conditions

Software testing and verification

Kinds of test:
  • Acceptance testing:
  • Integration testing:
  • Unit testing:
  • Functional testing:
  • Performance testing:
  • Regression testing:
  • Stress testing:
  • Usability testing:
  • Verifying whether the whole system works as intended.
  • Ensuring that software components or functions operate together.
  • Validating that each software unit performs as expected.
  • Checking functions by emulating business scenarios, based on functional requirements.
  • Testing how the software performs under different workloads.
  • Checking whether new features break or degrade functionality.
  • Testing how much strain the system can take before it fails.
  • Validating how well a customer can use a system or web application to complete a task.
Software testing is done to detect:
  • Architectural flaws
  • Poor design decisions
  • Invalid or incorrect functionally
  • Security vulnerabilities
  • Scalability issues
Testing best practices:
  1. Continous Testing
  2. Configuration management
  3. Servic virtualization
  4. Defect or "bug tracking"
  5. Metrics and reporting
  1. Project teams test each build as it becomes available. This type of software testing relies on test automation that is integrated with the deployement process.
  2. Organizations centrally maintain test assets and track what software builds to test.
  3. Testing environments may not be available, especially early in code development. Service virtualization simulates the services and systems that are missing or not yet completed, enabling teams to reduce dependencies and test sooner.
  4. Monitoring defects is important to both testing and development teams for measuring and improving quality.
  5. Reporting and analytics enable team members to share status, goals and test results. Advanced tools integrate project metrics and present results in a dashboard.
